    Implementing a driver: a practical example for a our LPUART peripheral in C

    This section explains how we crafted a custom driver for our emulated LPUART model, which we discussed the implementation in this document.

    ℹ️ A shortcut

    As a sort of shortcut, we also suggest downloading the vendor's SDK and diving into the example files. Given that the examples are meant to be run on the physical board, their drivers are rather complex, but with a bit of abstraction, you could easily get to understand the core functionality and behavior of the peripheral. Additionally, these examples provide mapped addresses and useful masks for extracting information from registers, making the role of each register much clearer and helping you to better understand how the hardware interacts.

    1. Register Mapping

    The LPUART peripheral registers are memory-mapped and defined as follows: The values are defined according to S32K3X8 Reference Manual.

    /* Register Map */
#define VERID                   0x0   /**< Version ID */
#define GLOBAL                  0x8   /**< Global */
#define BAUD                    0x10  /**< Baud Rate */
#define STAT                    0x14  /**< Status */
#define CTRL                    0x18  /**< Control */
#define DATA                    0x1C  /**< Data */
#define FIFO                    0x28  /**< FIFO */

    Each register is memory-mapped using the base address of the LPUART peripheral and offsets:

    #define LPUART_BASE_ADDR 0x40328000 // LPUART Base Address

#define LPUART_VERID   (*(volatile uint32_t *)(LPUART_BASE_ADDR + VERID))
#define LPUART_GLOBAL  (*(volatile uint32_t *)(LPUART_BASE_ADDR + GLOBAL))
#define LPUART_BAUD    (*(volatile uint32_t *)(LPUART_BASE_ADDR + BAUD))
#define LPUART_STAT    (*(volatile uint32_t *)(LPUART_BASE_ADDR + STAT))
#define LPUART_CTRL    (*(volatile uint32_t *)(LPUART_BASE_ADDR + CTRL))
#define LPUART_DATA    (*(volatile uint32_t *)(LPUART_BASE_ADDR + DATA))
#define LPUART_FIFO    (*(volatile uint32_t *)(LPUART_BASE_ADDR + FIFO))

    2. Initializing the LPUART Peripheral

    The LPUART peripheral is initialized by configuring the control registers:

    void LPUART_Init(void) {
    // Reset global registers
    LPUART_GLOBAL |= LPUART_GLOBAL_RESET;

    // Set baud rate (must be properly configured based on clock)
    LPUART_BAUD = 0x0F000004; // Example: 9600 baud rate

    // Configure control register
    LPUART_CTRL |= CTRL_TE | CTRL_RE; // Enable transmitter and receiver
    LPUART_CTRL &= ~(CTRL_M | CTRL_M7); // Set 8-bit data mode
    LPUART_BAUD &= ~(BAUD_M10 | BAUD_SBNS); // Set 1 stop bit mode

    // Configure FIFO
    LPUART_FIFO |= LPUART_FIFO_0to1_RESET;

    // Clear possible errors
    LPUART_STAT &= ~(STAT_FE | STAT_OR);
}

    3. Sending and Receiving Data

    The following functions implement basic data transmission and reception over LPUART.

    Sending a Byte

    void LPUART_SendByte(uint8_t data) {
    // Wait until the transmit FIFO is ready
    while (!(LPUART_STAT & STAT_TDRE));

    // Write data to the DATA register
    LPUART_DATA = data;
}

    Receiving a Byte

    uint8_t LPUART_ReceiveByte(void) {
    // Wait until data is available in the receive FIFO
    while (!(LPUART_STAT & STAT_RDRF));

    // Read data from the DATA register
    return (uint8_t)LPUART_DATA;
}

    4. Sending a String

    A helper function to send a string over LPUART:

    void LPUART_SendString(const char *str) {
    while (*str) {
        LPUART_SendByte((uint8_t)(*str));
        str++;
    }
}

    5. Printing Integers

    A simplified function to output an integer as a string:

    void LPUART_print_int(int value) {
    char buffer[12];
    char *ptr = buffer + sizeof(buffer) - 1;
    *ptr = '\0';

    if (value == 0) {
        *(--ptr) = '0';
    } else {
        bool is_negative = (value < 0);
        if (is_negative) {
            value = -value;
        }
        while (value > 0) {
            *(--ptr) = '0' + (value % 10);
            value /= 10;
        }
        if (is_negative) {
            *(--ptr) = '-';
        }
    }
    while (*ptr) {
        LPUART_SendByte(*ptr++);
    }
}

    6. Implementing printf-like Functionality

    A simple formatted print function that supports %d (integer) and %c (character):

    void LPUART_printf(const char *s, ...) {
    va_list args;
    va_start(args, s);

    while (*s != '\0') {
        if (*s == '%' && (*(s + 1) == 'd' || *(s + 1) == 'c')) {
            char specifier = *(s + 1);
            s += 2;

            if (specifier == 'd') {
                int value = va_arg(args, int);
                LPUART_print_int(value);
            } else if (specifier == 'c') {
                char value = (char)va_arg(args, int);
                LPUART_SendByte(value);
            }
        } else {
            LPUART_SendByte(*s);
            s++;
        }
    }
    va_end(args);
}

    Summary

    With these functions, the LPUART peripheral implemented in QEMU can be effectively used in an embedded environment. Additional utility functions like LPUART_SendString and LPUART_printf provide high-level abstraction for easier usage.
