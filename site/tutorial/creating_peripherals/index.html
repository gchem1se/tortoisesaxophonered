    Adding a Custom Peripheral to a SoC in QEMU

    The following six steps guide you through configuring and implementing the peripheral within QEMU.

    1. Adding the Peripheral to Kconfig

    QEMU requires that each new peripheral be explicitly defined in the build configuration. To do so:

    • Open qemu/hw/arm/Kconfig
    • Add the following configuration line to register your peripheral:
    config S32K3X8_USART
    bool

    This declares the new peripheral so it can be enabled during compilation.

    2. Adding the Peripheral to the SoC Configuration

    To ensure the peripheral is included in the SoC configuration:

    • Open qemu/hw/char/Kconfig
    • Modify the existing SoC definition to include the new peripheral:
    config S32K3X8_SOC
    bool
    select ARM_V7M
    select S32K3X8_USART

    This links the custom peripheral to the SoC, ensuring it is part of the compiled SoC definition.

    3. Registering the Peripheral in the Build System

    To integrate the peripheral into the QEMU build system:

    • Open qemu/hw/arm/meson.build
    • Add the following line:
    system_ss.add(when: 'CONFIG_S32K3X8_USART', if_true: files('s32k3x8_usart.c'))

    This ensures the source file for the peripheral is compiled when the S32K3X8_USART configuration option is enabled.

    4. Implementing the Peripheral (s32k3x8_usart.c)

    The new peripheral should be implemented as a C source file inside qemu/hw/char/. The file should include:

    • Initialization functions: Define how the peripheral is instantiated and registered in QEMU.
    • Read/Write functions: Handle access to memory-mapped registers.
    • Interrupt handling: Manage IRQs triggered by the peripheral.
    • QEMU integration: Use QEMU’s internal macros and functions for device emulation.

    The following header files should be included:

    #include "qemu/osdep.h"
#include "hw/char/s32k3x8_lpuart.h"
#include "hw/irq.h"
#include "hw/qdev-properties.h"
#include "hw/qdev-properties-system.h"
#include "qemu/log.h"
#include "qemu/module.h"

    A sample implementation should include functions for:

    • Checking if the peripheral can receive data (s32k3x8_usart_can_receive)
    • Receiving data (s32k3x8_usart_receive)
    • Resetting the peripheral (s32k3x8_usart_reset)
    • Handling register reads (s32k3x8_usart_read)
    • Handling register writes (s32k3x8_usart_write)
    • Registering memory-mapped regions
    • Registering IRQs

    5. Defining Memory-Mapped Registers

    The peripheral should expose registers through a memory-mapped interface. This is done using MemoryRegionOps in QEMU:

    static const MemoryRegionOps s32k3x8_usart_ops = {
    .read = s32k3x8_usart_read,
    .write = s32k3x8_usart_write,
    .endianness = DEVICE_NATIVE_ENDIAN,
};

    The read and write functions handle access to registers based on the given offset (addr). These functions update internal state variables and trigger appropriate actions, such as setting flags or sending interrupts.

    6. Registering the Peripheral Type in QEMU

    Finally, the peripheral must be registered as a QEMU type so it can be instantiated in a SoC.

    • Define the peripheral’s initialization functions:
    static void s32k3x8_usart_init(Object *obj) {
    S32K3X8LpuartState *s = S32K3X8_USART(obj);
    sysbus_init_irq(SYS_BUS_DEVICE(obj), &s->irq);
    memory_region_init_io(&s->mmio, obj, &s32k3x8_usart_ops, s, TYPE_S32K3X8_LPUART, 0x400);
    sysbus_init_mmio(SYS_BUS_DEVICE(obj), &s->mmio);
}
    • Define how the device is realized:
    static void s32k3x8_usart_realize(DeviceState *dev, Error **errp) {
    S32K3X8LpuartState *s = S32K3X8_USART(dev);
    qemu_chr_fe_set_handlers(&s->chr, s32k3x8_usart_can_receive,
                             s32k3x8_usart_receive, NULL, NULL,
                             s, NULL, true);
}
    • Register the peripheral type:
    static const TypeInfo s32k3x8_usart_info = {
    .name          = TYPE_S32K3X8_LPUART,
    .parent        = TYPE_SYS_BUS_DEVICE,
    .instance_size = sizeof(S32K3X8LpuartState),
    .instance_init = s32k3x8_usart_init,
    .class_init    = s32k3x8_usart_class_init,
};

static void s32k3x8_usart_register_types(void) {
    type_register_static(&s32k3x8_usart_info);
}

 type_init(s32k3x8_usart_register_types)

    This process ensures that the peripheral is recognized as a valid QEMU device and can be instantiated in an emulated SoC.

    That's it! You have successfully added a custom peripheral to QEMU.
