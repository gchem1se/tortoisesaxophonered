    Morse code decoder implementation

    To demonstrate the functionality of the implemented peripherals and drivers, we developed an example application to decode Morse code characters on the embedded system.

    ℹ️ Note

    for information about how to compile the code refer to Getting Started

    How It Works?

    The user interacts with the emulated embedded board via a keyboard, where the dot (".") key is used for input. Upon pressing the key, a corresponding LED is activated for a duration proportional to the length of the key press. The system interprets the duration of the LED activation as follows:

    • A DOT is recognized if the LED remains on for a duration between 0.1 and 1.4 seconds.
    • A HYPHEN is interpreted if the LED remains on for a duration of 1.5 seconds or longer.

    Once all the Morse code signals have been entered, the character can be decoded by pressing the D key.

    As defined by the Morse code specification, it is not possible to input more than 5 characters.

    How Is It Implemented?

    Is possible to find the implementations of the example in the folder example_morse_code with the sturcture represented below: 

    example_morse_code/  
├── FreeRTOSConfig.h        # necessary configurations for FreeRTOS        
├── gpio.c                  # implementations of a mocked gpio
├── gpio.h  
├── linker.ld     
├── main.c                  # implementations of the decode task
├── morse_code.c            # functions to decode the morse code
├── morse_code.h
├── startup.c               
└── Makefile

    Starting to analyze the principal file, main.c:

    1. The first step is to create a task that receives and decodes the Morse code, then schedules the corresponding actions.
    2. The task receives input through LPUART_ReceiveByte, capturing the characters to decode, and records the start time when the user begins pressing the key. This is done using startTime = xTaskGetTickCount();, and the LED is powered on.
    3. Since the key continues to send the pressed character at a specific frequency, to detect when the key is released, we use the GPIO_CheckKeyRelease() function.

    4. The function continues receiving characters until the current character differs from the previous one or until no character is received for a time period greater than 600 ticks:

      ⚠️ WARNING: The time range may vary depending on the device.

      uint8_t GPIO_CheckKeyRelease(void) {

    volatile uint32_t start = xTaskGetTickCount();
    while (!(LPUART_STAT & STAT_RDRF)) {
        // WARNING: THE TIME MAY BE DIFFERENT IN DIFFERENT DEVICES
        if ((xTaskGetTickCount() - start) * portTICK_PERIOD_MS > 600) {
            LED_Off();
            return '\0';
        }
    }
    return (uint8_t)LPUART_DATA;
}

    5. After all the required characters have been entered and the "D" key is pressed, the task calls the function decodeMorse(buffer) to decode the Morse code. This function is implemented in morse_code.c.
