    Getting Started

    This guide provides all the necessary steps to build and run our project components.

    Building the Documentation

    To build and view the documentation locally, you'll need to install the required Python packages:

    pip install mkdocs mkdocs-material

    Then, from the root of the project:

    • For local development with live reload: mkdocs serve
    • To build the static site: mkdocs build

    Building QEMU

    Prerequisites

    Before building QEMU, ensure you have all required packages installed. Please refer to our building requirements guide for a complete list of dependencies.

    Build Process

    1. Navigate to the qemu folder

    2. Configure the build: 

      ./configure --target-list=arm-softmmu
      Note: The --target-list flag is optional and will only build ARM targets, speeding up the build process.

    3. Build QEMU: 

      cd build
make -j

    ℹ️ Note

    To optimize build times during development, we've disabled most unused board configurations. For details about which files were modified to achieve this, please refer to our building tutorial.

    Building the Examples

    Prerequisites

    Install the ARM cross-compilation toolchain: 

    sudo apt install gcc-arm-none-eabi
    We've tested with arm-none-eabi-gcc version 10.3 from APT and encountered no issues. The toolchain distributed by APT corresponds to the one indicated by the vendor (NXP) to build code for the target architecture.

    Build Process

    For each example (example_led or example_morse_code):

    1. Navigate to the example directory
    2. Build the example: 
      make
    3. Run the example: 
      make run
      Or for debugging: 
      make debug

    Key Implementation Files

    Our custom implementations of emulated HW inside of QEMU can be found in the following files:

    • qemu/include/hw/arm/s32k3x8_soc.h
    • qemu/hw/arm/s32k3x8_soc.c
    • qemu/hw/arm/nxp_s32k3x8evb.c
    • qemu/hw/char/s32k3x8_lpuart.c
    • qemu/include/hw/char/s32k3x8_lpuart.c

    Detailed documentation about them is to be found in the Tutorial section of this documentation, which features our implementations as leading examples.
